Projeto XOPS · CI/CD Inteligente com IA

Sobre o projeto

Este projeto é focado em conceber, implementar e avaliar uma pipeline CI/CD inteligente com quality gates potenciados por IA.

Visão geral

A aplicação é uma pequena API em Node.js/Express que serve de base para o pipeline. O foco não é a complexidade funcional, mas sim a qualidade do código e segurança da pipeline.

O que a pipeline faz?

  • Compila e instala dependências do projeto.
  • Corre testes automáticos com Jest + Supertest.
  • Gera relatórios de cobertura (lcov / clover).
  • Usa IA para revisão de código (quality gate).
  • Usa IA para SAST – análise estática de segurança.

IA integrada no pipeline

O projeto integra ferramentas baseadas em LLM diretamente na pipeline. Em vez de a IA ser “um extra”, ela faz parte do processo de decisão sobre se o commit é aceite ou não.

Quality gates com IA

  • ai-review.js – analisa o código e devolve um score de qualidade, nível (OK / WARNING / CRITICAL) e comentários de melhoria.
  • ai-sast.js – faz SAST, procura vulnerabilidades comuns e recomenda boas práticas de segurança.
  • Se o nível for CRITICAL ou o score ficar abaixo do limite, o build é reprovado.

Pipeline CI/CD do XOPS

A pipeline é definida em .github/workflows/main.yml e corre automaticamente em cada push ou pull request.

Etapas principais

  • Checkout do repositório.
  • Configuração do Node.js e cache de dependências.
  • Instalação de pacotes com npm ci.
  • Execução de testes unitários com cobertura.
  • Upload do relatório de cobertura como artefacto.
  • Execução do npm run ai-review.
  • Execução do npm run ai-sast (security gate).
GitHub Actions Testes + Cobertura Artefactos de build

Stack & Endpoints

A API expõe poucos endpoints, suficientes para demonstrar testes, cobertura e integração com a IA.

Stack técnica

  • Node.js + Express para a API.
  • Jest + Supertest para testes.
  • dotenv para variáveis de ambiente.
  • OpenAI para code review e SAST.

Endpoints da API

  • GET / – mensagem simples “Hello from Sprint 13”.
  • GET /users – lista de utilizadores em JSON (usado nos testes).
  • GET /about – esta página HTML de apresentação do projeto.
